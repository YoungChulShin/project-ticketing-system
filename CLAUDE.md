# Project Ticketing System

공연 티켓 예매 대기열 시스템 학습 프로젝트.

## 진행 방식 (중요)

이 프로젝트는 학습이 목적이다. 설계·구현 모두 아래 사이클로 진행한다:

1. 작업 단위(커밋 하나) 시작 전에 무엇을 왜 만드는지, 새로 등장하는 개념을 먼저 설명한다
2. 코드 작성 후 핵심 결정 포인트를 해설한다
3. 실행/테스트로 동작을 같이 확인한다 (실행은 가능하면 사용자가 직접)
4. 커밋 전에 변경 사항을 설명하고 사용자 확인을 받는다. 확인 없이 커밋하지 않는다
5. 질문을 받고, 사용자 확인 후 다음 단위로 넘어간다

한 번에 여러 작업 단위를 몰아서 진행하지 않는다.

### 사용자 직접 구현

백엔드 핵심 로직 일부는 사용자가 직접 작성한다 (사용자가 학습자다):

- Claude가 요구사항·인터페이스·힌트를 제시하고, 사용자가 구현한다
- 사용자 구현 후 Claude가 리뷰하고 피드백한다. 정답을 바로 주지 않고 스스로 고칠 기회를 먼저 준다
- 대상은 학습 가치가 높은 부분 (예: Lua 스크립트, 대기열 서비스 로직). 보일러플레이트·설정·프론트엔드는 Claude가 작성한다
