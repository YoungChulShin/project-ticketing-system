# Project Ticketing System

공연 티켓 예매 대기열 시스템 학습 프로젝트.

## 진행 방식 (중요)

이 프로젝트는 학습이 목적이다. 설계·구현 모두 아래 사이클로 진행한다:

모든 작업은 이 사이클을 따른다 — **진행 전 알림 → 진행 → 완료 후 설명 → 사용자 확인 → 커밋**:

1. 진행 전: 무엇을 왜 할지 먼저 알린다 (새 개념 설명 포함). 알리지 않은 작업을 먼저 시작하지 않는다
2. 진행: 코드 작성/명령 실행
3. 완료 후: 무엇이 어떻게 됐는지, 핵심 결정 포인트를 설명한다 (실행 확인은 가능하면 사용자가 직접)
4. 사용자 확인을 기다린다
5. 커밋: 변경 사항 요약 + 승인 후에만 커밋한다

디버깅도 예외 아님 — 예상 밖 문제 발견 시 원인과 해결책을 먼저 알리고, 특히 repo 밖(홈 디렉토리, 전역 설정)을 건드리는 해결책은 반드시 사전 확인을 받는다.
한 번에 여러 작업 단위를 몰아서 진행하지 않는다.

### 사용자 직접 구현

백엔드 핵심 로직 일부는 사용자가 직접 작성한다 (사용자가 학습자다):

- Claude가 요구사항·인터페이스·힌트를 제시하고, 사용자가 구현한다
- 사용자 구현 후 Claude가 리뷰하고 피드백한다. 정답을 바로 주지 않고 스스로 고칠 기회를 먼저 준다
- 대상은 학습 가치가 높은 부분 (예: Lua 스크립트, 대기열 서비스 로직). 보일러플레이트·설정·프론트엔드는 Claude가 작성한다
